Docker基本使用

基本命令

  1. docker run -it ubuntu :使用ubuntu镜像，本地没有且未指定版本，默认从Docker Hub拉取latest版本
    • -i表示STDIN开启，
    • -t表示为分配的容器分配伪tty终端
    • -d表示容器在后台运行
    • –name指定容器名字
    • –restart由于错误容器停止时，检查错误代码决定是否重启容器
  2. docker ps:查看正在运行的容器
    • -a查看所有容器
    • -q只返回容器id
  3. docker start:启动容器
  4. docker attach:附着到正在运行的容器
  5. docker logs:查看容器日志
    • -f:类似于tail -f
    • –tail:跟踪日志片段
    • -t:为日志附加时间戳
  6. docker stop:停止容器
  7. docker rm:删除容器
    • -f:删除运行的容器
  8. docker rm ‘docker ps -a -q’:删除所有容器
  9. docker top:查看容器内的进程
  10. docker stats:查看容器的状态，可同时指定多个容器
  11. docekr exec:在容器内部启动额外的新进程，docker exec -it container_id bash比较常用
  12. docker inspect:查看容器详细信息，包括配置信息、网络信息等
  13. docker images:列除docker镜像
  14. docker pull:从docker hub拉取镜像
  15. docker search:查找docker hub上的镜像
  16. docker commit:创建对象，只创建容器镜像和当前容器的差异部分，轻量化更新
    • -m指定提交信息
    • -a指定作者
  17. docker build:构建镜像 docker build -t=”name” -f filePath
    • -t设置仓库和名称，默认标签为latest
    • -f指定构建源的位置
    • –no-cache:不使用缓存，确保每次构建都使用最新内容

镜像的基本使用

镜像由文件系统叠加而成
最低层是引导文件系统bootfs，容器启动后会被卸载
第二层是root文件系统rootfs，rootfs可以是一种或多种操作系统，永远为只读状态
一个镜像由多个镜像叠加而成，启动时docker会在最顶层添加一个读写层
docker使用写时复制机制，当文件系统发生变化时，从下层的只读层复制到顶层读写层，只读版本依然存在，但被副本所隐藏

Dockerfile构建镜像
  • 每条RUN都会创建一个新镜像层，执行成功，会提交该层
  • 某条失败后，会保存中间镜像层，便于调试定位
